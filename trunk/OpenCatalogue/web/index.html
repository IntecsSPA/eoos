Earth Observation Open Search Catalogue

Get the latest version on Google.

OpenSearch was conceived by A9.com, an Amazon.com company, as a mechanism to trigger a distributed search over a collection of web sites and merge the results in an unique result set (search results syndication).

The entry point to a search engine is its description document. This document contains, along with other metadata, an URL template for each accepted response format. Each template indicates the mandatory and optional query parameters, as well as the syntax to build the query as an URL. The query is performed as an HTTP GET.

Based on OGC standards

The EOOS catalogue manages metadata compliant with OGC 10-157r4, EO Profile of Observations and Measurements. The search interface is based on OpenSearch and support the Geotemporal Extension for OpenSearch (OGC 10-032r7) and the OpenSearch Extension for Earth Observation (OGC 13-026r2).

Search Data

Search

Searches can be done using the OpenSearch standard with the Geotemporal Extension for OpenSearch (OGC 10-032r7) and the OpenSearch Extension for Earth Observation (OGC 13-026r2). See some examples. The entry point of the catalogue is the OpenSearch Description Document (OSDD) which provides metadata about the contents along with a set of URL Templates which inlustrate the parameters accepted by the service and the variety of output formats in which the results can be retrieved.

Ingest Custom Metadata

Data can be ingesed in the system via a converter. The converter accept a csv or an XML file as input and generate O&M compliant metadata according to a configuration file (to be generated offline and configured in the system). The configuration file defines the mapping between the columns of the csv file or the XPATH of the source file and the elements of the O&M metadata file.
